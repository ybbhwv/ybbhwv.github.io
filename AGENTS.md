# 仓库指南

## 项目结构与模块组织
本仓库目前以单静态页面的形式交付，文件位于 `index.html`。该文件包含了 HTML 标记、Tailwind 工具类、`<style>` 块中的自定义 CSS，以及位于行内 `<script>` 中的 Vue 3 应用逻辑。

请按功能关注点对相关更改进行分组：
- **页面结构与内容**：`index.html` 中间和顶部的 HTML 部分。
- **视觉样式**：`<style>` 块和 Tailwind 类。
- **交互行为与数据**：靠近底部的 Vue `setup()` 块。

如果页面规模扩大，在添加更多功能之前，请先将资源拆分至 `assets/` 文件夹，样式拆分至 `styles/`，应用逻辑拆分至 `scripts/`。

## 构建、测试与开发命令
目前未配置构建步骤。开发基于文件直接进行。

- `python -m http.server 8000`
  在仓库根目录下运行简易本地服务器，用于浏览器测试。
- `start http://localhost:8000`
  启动服务器后，在 Windows 浏览器中打开本地页面。
- `git status`
  在提交前检查工作树状态。

除非仓库首先引入了真正的多文件前端结构，否则请避免引入打包工具（Bundler）或包管理器。

## 代码风格与命名规范
HTML、CSS 和 JavaScript 均使用 **4 空格缩进**，以保持与现有文件一致。首选语义化的部分 ID，例如 `home`、`memories` 和 `timeline`。此外请遵循：

- Vue 的 ref、响应式对象和函数使用 `camelCase`（小驼峰命名法）。
- 代码内部使用清晰的**英文标识符**，即使页面可见内容是中文。
- 采用描述性的 Tailwind 类分组，将布局类置于颜色/特效类之前。

保持自定义 CSS 最小化，并将可复用的 Token 放置在 `:root` 中。

## 测试指南
目前尚未建立自动化测试套件。对于每次更改：

- 在浏览器中加载页面，验证布局、倒计时逻辑和表单行为。
- 使用开发者工具检查桌面端和移动端的宽度适配。
- 如果 Vue、Tailwind 或 Font Awesome 的 CDN 地址发生变更，请重新测试外部依赖项。

后续添加测试时，请将其置于 `tests/` 目录下，并命名为 `*.test.js`。

## 提交与拉取请求（PR）指南
Git 历史目前使用简短直接的提交信息，如 `初始化`。请继续使用简洁、聚焦于目的的主题（中文或英文均可），例如 `修复倒计时日期计算` 或 `Extract timeline data`。

拉取请求（Pull Request）应包含：
- 用户可见更改的简要总结。
- 针对 UI 更新的屏幕截图或录屏。
- 关于已执行手动验证的说明。
- 若适用，请链接相关的 Issue 或任务 ID。
